## v0.19.0 - 2026-05-07
### Added
* Acquisition support for locked files on NTFS
* GPT boot support for QCOW images
* Additional directory and history shellitem id
* Cargo profiling profile
* Support for beef0003 shellitem
* Artifact output files to JSON report
* Support for reading remote yara rules
* ESXi support
* Option to exclude directories for filelisting
* Include drive (if available) to MFT output
* Additional report values: commit hash, feature flags, arguements
* Entitlements for signed macOS binaries (required for Yara-X)
### Changed
* Change journals artifact to journal for artemis cli
* Added more descriptive artifact name for reports when running scripts
* Overhaul for file acquisitions
* Reduced memory usage when parsing usnjrnl
* Improved macOS bookmark parsing
* Major improvements when parsing Windows Schedule Tasks
* Reduced clone usage
* Improved performance of outputting data locally
* Made csv output nicer to review
* Reduced memory usage of eventlogs parser
* Can apply either directory and/or filename regex filtering for filelisting
* Made all cli arguements consistent. --alt-file when you want to provide an alternative file. --alt-dir when you want to provide an alternative directory. Removed --alt-path.
* Made shortcut and jumplists artifacts accept globs are arguements
* Renamed report version field to artemis_version
* Made output consistent when an artifact has no results. Now if artemis parses an artifact that has no data, it will just output runtime log and a JSON report. Previously artemis would output an empty result with runtime metadata
### Fixed
* Issue where artemis did not fully timeline usnjrnl
* Bug where artemis may not parse all journal log files
* Incorrect service state and sid_type values
* Issue where windows service state was not correct
* Bug where artemis would not properly parse Schedule Tasks with HTML entities
* Bug where artemis would skip binary message blobs in systemd journal
* Issue where artemis would follow symbolic links when hashing files
* Issue where artemis would not parse some Control Panel Shellbag entries
### Dependencies
* Switched from rand to fastrand
* Added windows-sys crate
* Added rust-crypto dependencies: base16ct and digest-io
* Updated all dependencies to latest version
### Tests
* Improvements to integration tests
